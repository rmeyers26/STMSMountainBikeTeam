# Plan: Add CSV Export Option to Apparel Report

## Overview
Add a "Export CSV" button next to the existing "Print Report" button in the admin orders page (admin-orders.html). This will allow admins to download the apparel order data as a CSV file for external analysis or reporting.

## Current State
- The page already has a "🖨️ Print Report" button that triggers `window.print()` for printing the table.
- Order data is displayed in a dynamic HTML table with headers, data rows, and a subtotal footer row.
- Table includes columns: Submitted, First, Last, Email, Phone, Type, #, Rider Name, Gender, Role, Team, Race Qty, Race Size, Bot. Type, Bot. Qty, Bot. Size, Prac Qty, Prac Size.

## Implementation Steps

1. **Add Export CSV Button**
   - Insert a new button element with id `admin-export-csv-btn` in the button group div, after the print button.
   - Style: `class="btn btn-primary"` with icon 📊
   - Text: "Export CSV"

2. **Implement CSV Export Functionality**
   - Add event listener for the new button in the JavaScript section.
   - Extract data from the existing HTML table (`.admin-report-table`).
   - Convert table rows to CSV format:
     - Include all table rows (headers, data, and subtotal).
     - Properly escape cell content for CSV (wrap in quotes, escape internal quotes).
     - Use comma separation, newline for rows.
   - Generate a downloadable CSV file named `apparel-orders.csv`.
   - Handle case where no table exists (no data to export).

3. **Testing Considerations**
   - Verify CSV opens correctly in spreadsheet applications (Excel, Google Sheets).
   - Ensure special characters and formatting are preserved.
   - Test with various data volumes (empty table, single row, multiple rows).
   - Confirm no security issues (data is already displayed on page, just different format).

## File Changes
- **admin-orders.html**: Add button HTML and JavaScript event handler for CSV export.

## Dependencies
- No external libraries required - uses native browser Blob and download APIs.
- Compatible with existing code structure and session authentication.

## Security Notes
- CSV export only accessible to authenticated admins (same as page access).
- No additional server-side changes needed.
- Data is client-side generated from already-displayed table data.
